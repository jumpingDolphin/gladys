# Gladys — Personal AI Assistant

A personal AI assistant built on [OpenClaw](https://openclaw.dev), accessed via Telegram (text + voice), with Claude as the primary reasoning model. Runs on a Hetzner VPS (bare metal, no Docker).

## Repository layout

This repo (`~/gladys`) contains project files at the root and OpenClaw state in the `openclaw/` subfolder (`OPENCLAW_STATE_DIR=~/gladys/openclaw`).

```
~/gladys/                              # git repo
├── .env.example                       # secret template (no values)
├── .env                               # actual secrets (gitignored)
├── .gitignore
├── README.md
├── CLAUDE.md
├── roadmap.md                         # full project roadmap
├── scripts/
│   └── setup.sh                       # VPS bootstrap: user, node, openclaw, systemd, ufw
└── openclaw/                          # OPENCLAW_STATE_DIR (generated by openclaw onboard)
    ├── openclaw.json                  # config (generated by wizard)
    ├── agents/
    │   └── main/
    │       ├── agent/                 # IDENTITY.md, MEMORY.md
    │       ├── memory/                # daily context (*.md backed up, *.sqlite gitignored)
    │       └── sessions/              # gitignored (private transcripts)
    ├── credentials/                   # gitignored
    ├── sandboxes/                     # gitignored
    └── extensions/                    # gitignored
```

## Phase 0: VPS Setup

### Prerequisites

- A fresh Ubuntu VPS (Hetzner) with root SSH access
- DNS pointing to the VPS (e.g. `gladys.simonschenker.com`)
- A GitHub deploy key will be generated during setup

### Steps

1. **SSH to the VPS as root:**
   ```
   ssh root@gladys.simonschenker.com
   ```

2. **Get `setup.sh` onto the server** (e.g. via `scp` or paste):
   ```
   scp scripts/setup.sh root@gladys.simonschenker.com:/root/setup.sh
   ```

3. **Run the bootstrap script:**
   ```
   bash /root/setup.sh
   ```
   This installs Node.js, OpenClaw, Claude Code, creates user `simon`, hardens SSH, configures UFW, clones the repo, and sets up the systemd service. When prompted, add the deploy key to GitHub.

4. **SSH in as simon:**
   ```
   ssh simon@gladys.simonschenker.com
   ```

5. **Fill in API keys:**
   ```
   nano ~/gladys/.env
   ```

6. **Run the OpenClaw onboard wizard** (do NOT use `--install-daemon` — the systemd unit is already installed by `setup.sh`):
   ```
   openclaw onboard
   ```

7. **Review and commit the generated config:**
   ```
   git add openclaw/ && git diff --cached
   git commit -m "Add OpenClaw config via wizard"
   ```

8. **Start the gateway:**
   ```
   systemctl --user start openclaw-gateway
   ```

9. **Verify:**
   ```
   openclaw doctor
   systemctl --user status openclaw-gateway
   ```

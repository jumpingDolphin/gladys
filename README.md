# Gladys — Personal AI Assistant

A personal AI assistant built on [OpenClaw](https://openclaw.dev), accessed via Telegram (text + voice), with Claude Opus 4.6 as the primary reasoning model. Runs on a Hetzner VPS (bare metal, no Docker) at `gladys.simonschenker.com`.

## VPS setup (Phase 0 — complete)

### Prerequisites

- A fresh Ubuntu VPS (Hetzner) with root SSH access
- DNS pointing to the VPS (e.g. `gladys.simonschenker.com`)
- A GitHub deploy key will be generated during setup

### Steps

1. **SSH to the VPS as root:**
   ```
   ssh root@gladys.simonschenker.com
   ```

2. **Get `setup.sh` onto the server** (e.g. via `scp` or paste):
   ```
   scp scripts/setup.sh root@gladys.simonschenker.com:/root/setup.sh
   ```

3. **Run the bootstrap script:**
   ```
   bash /root/setup.sh
   ```
   Installs Node.js 22.x, OpenClaw, Claude Code, creates user `simon`, hardens SSH, configures UFW, clones the repo, and sets up the systemd service. When prompted, add the deploy key to GitHub.

4. **SSH in as simon:**
   ```
   ssh simon@gladys.simonschenker.com
   ```

5. **Fill in API keys:**
   ```
   nano ~/gladys/.env
   ```

6. **Run the OpenClaw onboard wizard** (do NOT use `--install-daemon` — the systemd unit is already installed by `setup.sh`):
   ```
   openclaw onboard
   ```

7. **Review and commit the generated config:**
   ```
   git add openclaw/ && git diff --cached
   git commit -m "Add OpenClaw config via wizard"
   ```

8. **Start the gateway:**
   ```
   systemctl --user start openclaw-gateway
   ```

9. **Verify:**
   ```
   openclaw doctor
   systemctl --user status openclaw-gateway
   ```

## Dashboard access

The gateway binds to loopback only. To access the web UI from your local machine:

```bash
ssh -N -L 18789:127.0.0.1:18789 simon@gladys.simonschenker.com
```

Then open `http://localhost:18789/` in your browser.
